RESTful & RESTful API

Introdução ao padrão REST e como construir APIs RESTful

O que é REST?

  • REST = Representational State Transfer
  • Um estilo de arquitetura para sistemas distribuídos
  • Popularizado por Roy Fielding em sua tese de doutorado (2000)

Como surgiu o REST?

  • Criado como parte do desenvolvimento da arquitetura da Web
  • Necessidade de padronizar a comunicação entre sistemas
  • Baseado nos princípios do HTTP

RESTful API

  • Uma API que segue os princípios REST
  • Utiliza HTTP como protocolo de comunicação
  • Realiza operações com os métodos: GET, POST, PUT, DELETE

EndPoints

URL que representa um recurso

GET /api/usuarios
POST /api/usuarios
GET /api/usuarios/1
PUT /api/usuarios/1
DELETE /api/usuarios/1

Cabeçalhos (Headers)

  • Authorization: token ou credenciais
  • Content-Type: tipo do conteúdo (ex: application/json)
  • Accept: tipo de resposta esperada

Códigos de Status (HTTP)

  • 200 OK: Requisição bem-sucedida
  • 201 Created: Recurso criado
  • 400 Bad Request: Erro no cliente
  • 401 Unauthorized: Autenticação falhou
  • 404 Not Found: Recurso não encontrado
  • 500 Internal Server Error: Erro no servidor

Boas Práticas

  • Utilize substantivos nos endpoints (ex: /usuarios)
  • Use corretamente os métodos HTTP
  • Inclua mensagens de erro claras
  • Versão da API na URL (ex: /api/v1/)

REST x SOAP

  • REST: mais leve, usa JSON, baseado em HTTP
  • SOAP: mais pesado, usa XML, requer contratos (WSDL)
  • REST é mais popular para aplicações Web e Mobile

Exemplo de Requisição


curl -X POST https://api.exemplo.com/usuarios \
  -H "Content-Type: application/json" \
  -d '{"nome":"Maria","email":"maria@email.com"}'
